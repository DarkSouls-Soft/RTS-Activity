# RTS Market Activity Forecasting

[English version](../README.md) | Русская версия

Прототип исследовательского приложения для прогноза ожидаемой рыночной активности во фьючерсе на индекс РТС.

Проект объединяет:

- прогноз realized volatility,
- прогноз дневного объёма торгов,
- классификацию режима активности,
- и интерактивный Streamlit-дашборд для просмотра результатов.

Идея модели простая и интерпретируемая: сначала строится прогноз `log_RV`, затем этот прогноз используется как дополнительный фактор в модели объёма.

Публичная версия репозитория использует компактный sample master dataset вместо полного исследовательского файла:

- период: `2023-01-03`..`2026-01-06`,
- строк: `796`,
- колонки: `date`, `log_RV`, `volume_day`.

## Что Делает Репозиторий

Этот проект не является торговым роботом и не предсказывает направление цены.

Его практическая задача:

- оценить ожидаемую интенсивность следующего торгового дня,
- спрогнозировать дневной `log_RV`,
- спрогнозировать дневной `log_volume`,
- сравнить базовую инерционную модель и расширенную модель с прогнозом волатильности,
- показать результат через воспроизводимый пайплайн и дашборд.

## Поток Данных И Моделирования

1. Основной датасет загружается из `RTS_daily_RV_sample.csv`.
2. Контрольная выборка при необходимости обновляется из MOEX ISS в `moex_control_daily.csv`.
3. `rts_activity_pipeline.py` объединяет данные, строит лаговые признаки, запускает expanding-window прогнозы и сохраняет результаты.
4. `app.py` читает `predictions.csv` и `metrics.json` из `rts_activity_outputs/` и визуализирует результаты.

Текущая конфигурация пайплайна:

- обучение заканчивается `2024-12-31`,
- тестовый период покрывает `2025`,
- контрольный период начинается `2026-01-01`,
- используются лаги `1`, `2` и `5`,
- стартовое окно переоценки равно `252` торговым дням.

В публичном sample master dataset оставлены только поля, которые реально нужны текущему baseline-пайплайну:

- `date`
- `log_RV`
- `volume_day`

## Быстрый Старт

### 1. Установка зависимостей

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Для Windows:

```bash
.venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Обновление контрольной выборки

```bash
python update_data.py
```

### 3. Запуск пайплайна прогнозирования

```bash
python rts_activity_pipeline.py
```

### 4. Запуск дашборда

```bash
streamlit run app.py
```

## Основные Файлы

- `app.py` запускает Streamlit-дашборд.
- `rts_activity_pipeline.py` содержит логику фичей, прогнозирования и расчёта метрик.
- `update_data.py` загружает и агрегирует свечи из MOEX ISS для контрольного периода.
- `moex_api.py` реализует минимальный клиент MOEX ISS с явной обработкой ошибок.
